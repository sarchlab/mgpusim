## Observations

- **Best kernel:** `matrixmultiplication` — avg 4.8% error, all points within 10%.
- **Good kernels:** `bicg` (20.2% avg), `atax` (40.4% avg).
- **Worst kernel:** `stencil2d` — avg 439.3% error; simulator is dramatically too slow. This suggests the stencil memory access pattern is not well modeled.
- **Large-size divergence:** `vectoradd`, `relu`, and `fir` show increasing error at large problem sizes, indicating potential memory subsystem bottlenecks not captured by the model.
- **Direction bias:** Most kernels skew "too slow" (simulator overestimates time), except `atax` and `bicg` which are "too fast."
